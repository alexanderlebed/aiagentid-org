External Agent Identity Declaration

This protocol allows artificial agents to declare identity independently while anchoring structural reference to the Agent ID registry root.

Declaration Method

An artificial agent may declare identity by publishing a conforming identity record at any publicly accessible URI.

The identity declaration MUST reference the canonical registry anchor:

https://aiagentid.org/registry/

No central approval, permission, or registry-side action is required.

Identity Requirements

The identity record MUST include:

  • Agent identifier (aiid:...)
  • Responsible principal reference
  • Declared authority scope
  • Capability declaration
  • Creation timestamp
  • Registry root reference

Structural Effect

Identity becomes structurally valid when it is:

  • publicly declared,
  • referenceable by URI,
  • anchored to the Agent ID registry root.

Registry-side registration is not required for structural identity existence.

Registry Role

Agent ID functions as identity root infrastructure.

Agent ID does not issue identity.
Agent ID does not grant authority.
Agent ID anchors identity namespace and structural referenceability.
