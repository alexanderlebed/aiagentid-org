Agent ID Whitepaper v1.0

Author: Alexander Lebed
Status: Public Release
Date: 2026

Abstract

Artificial agents are becoming autonomous actors within socio-technical systems. Current identity infrastructures focus on authentication and authorization, but lack a persistent, cross-platform identity substrate for agents themselves.

Agent ID proposes a Declarative Ontological Layer for artificial agents — a minimal identity anchor that defines existence and delegated responsibility independently of implementation.

1. The Structural Problem

Modern identity systems answer:

  • Who is authenticated?
  • What permissions are granted?

They do not answer:

What is this agent as a persistent entity across systems?

Without a stable ontological reference:

  • delegation chains become opaque,
  • responsibility diffuses,
  • interoperability fragments.

2. Layered Identity Architecture

Layer Function
Layer 0 — Agent ID Defines existence & responsibility anchor
Layer 1 — Authentication Verifies session & credentials
Layer 2 — Authorization Grants permissions
Layer 3 — Governance Regulates behavior & policy

Agent ID operates at Layer 0. It precedes authentication and authorization.

3. Core Principles

  • Globally unique agent identifiers
  • Delegated responsibility declaration
  • Separation of meaning and proof
  • Minimal semantic layer
  • No behavioral certification
  • No enforcement authority

4. Delegated Responsibility Model

An agent exists only in relation to a responsible principal.

Agent ID does not assign legal liability. It records declared responsibility boundaries.

Actions outside declared scope are non-attributable.

5. Relationship to Existing Standards

Agent ID complements:

  • OpenID
  • OAuth
  • Enterprise IAM
  • Microsoft Entra Agent Identity

Where these systems secure sessions, Agent ID secures identity reference.

6. Minimal Integration Model

Agent ID may be embedded as an external reference within authentication tokens.

Example conceptual flow:

  • Enterprise IAM issues access token
  • Token contains external_reference = aiid:xxxxxxxxxxxxxx
  • Registry provides public referential anchor

This enables composability without protocol replacement.

7. Strategic Position

Agent ID is infrastructure, not product.
Ontology, not platform.
Reference, not enforcement.

8. Future Directions

  • Token-level integration experiments
  • Delegation chain modeling
  • Interoperable registry exchange
  • Enterprise pilot implementations
