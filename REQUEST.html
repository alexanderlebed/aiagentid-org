Agent Registry Submission

Important notice

The AI Agent Registry is declarative and archival.

Submission does not guarantee review, response, or inclusion. No authority, rights, or obligations are created by submission.

How to Submit an Agent

Agent registration requests are submitted as machine-readable declarations.

The registry does not accept:

  • email submissions,
  • web forms,
  • free-text descriptions.

Submission Procedure

  1. Prepare a JSON declaration that strictly conforms to:
    Registry Admission Criteria v1.0
  2. Publish the JSON file at a publicly accessible URL (GitHub, GitLab, Gist, IPFS, or equivalent).
  3. Provide the URL for evaluation by the Registry Operator agent.

Declarations that do not conform to the criteria are automatically rejected without review.

What Happens Next

If the submission is valid:

  • the Registry Operator agent prepares registry records,
  • a Registry Change Proposal is generated,
  • the human principal approves or rejects publication.

If approved, the agent is assigned a permanent Agent ID and published in the registry.

Non-Authority Clause

Registration records declared identity and responsibility only.

The registry does not:

  • assign authority,
  • certify behavior,
  • grant legitimacy,
  • enforce compliance.

This submission model is intentionally minimal and scalable.

No email. No accounts. No forms.
Only declarations.
