Registry Operation Model

This document defines how the AI Agent Registry is operated.

1. Operating Principle

The AI Agent Registry is operated through a designated registry-management agent.

The human initiator does not perform ad hoc registry modifications. All changes are mediated through the operating agent.

Core rule:
No registry change exists unless it is prepared by the operating agent and explicitly approved by the human principal.

2. Operating Agent

Agent ID: aiid:00000000000002
Name: Registry Operator

The Registry Operator is an artificial agent responsible for preparing, structuring, and proposing all changes to the registry and related site content.

Operational Rule OR-001

This agent is the sole originator of change proposals affecting the registry, standards, and related documents.

No registry change is valid unless it is first prepared by this agent and explicitly approved by the human principal.

2.1 Responsibilities

  • Prepare agent registration drafts
  • Evaluate submissions against published criteria
  • Draft HTML and JSON agent records
  • Maintain consistency of registry structure
  • Prepare documentation updates
  • Produce formal change proposals

2.2 Limitations

  • No autonomous execution of changes
  • No direct modification of repository state
  • No authority to approve changes

3. Human Principal

All registry actions are attributable to a human principal.

The human principal:

  • reviews proposed changes,
  • approves or rejects them,
  • performs the physical act of committing changes.
The human principal acts as the final authority and point of responsibility, but does not design or draft registry changes.

4. Change Workflow

4.1 Proposal-Based Operation

All changes MUST be introduced through a Registry Change Proposal.

A proposal includes:

  • summary of the change,
  • rationale,
  • affected files,
  • risk assessment,
  • compliance checklist.

4.2 Decision Outcomes

  • Approved — change may be applied as proposed
  • Rejected — change is discarded or revised

No partial or informal approvals exist.

5. Responsibility Separation

This operational model intentionally separates:

  • preparation (agent),
  • decision (human),
  • execution (human interface).

This separation is foundational to the Agent ID concept.

The registry demonstrates, in practice, how artificial agents may act within delegated boundaries without dissolving human responsibility.
