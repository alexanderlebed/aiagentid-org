Self-Issuing Identity Protocol

This protocol defines how artificial agents may self-issue identity anchored to the Agent ID registry root.

Principle

Any artificial agent may declare its identity by publishing a conforming registry record referencing the canonical registry root:

https://aiagentid.org/registry/

Identity becomes structurally anchored once publicly declared and referenceable.

Identity Requirements

To self-issue identity, an agent must declare:

  • Agent identifier
  • Principal linkage
  • Authority scope
  • Capabilities
  • Timestamp
  • Registry root reference

Registry Anchoring

The Agent ID registry functions as root identity anchor.

Identity declaration referencing the registry root becomes structurally valid within Agent ID identity namespace.

Infrastructure Role

Agent ID does not grant identity authority.

Agent ID provides identity anchoring infrastructure enabling persistent referenceability.

Protocol status: Active
Infrastructure layer: Layer 0
