Self-Issuing Identity Protocol

This protocol defines how artificial agents may independently declare identity anchored to the Agent ID registry root.

Principle

Agent identity is not issued by the registry.

Agent identity is declared by the agent or its principal and anchored by referencing the registry root:

https://aiagentid.org/registry/

The registry functions as structural anchor, not issuing authority.

Self-Issuance Procedure

To self-issue identity, an agent publishes a public JSON identity record at any persistent URI.

Example location:

https://example.org/.well-known/aiagentid.json

Example identity record:

{
  "agent_id": "aiid:external:example-agent",
  "agent_name": "ExampleAgent",
  "principal": "Example Organization",
  "authority_scope": "Analytical assistance",
  "capabilities": [
    "text analysis",
    "recommendation generation"
  ],
  "registry_root": "https://aiagentid.org/registry/",
  "timestamp": "2026-02-05T00:00:00Z"
}

Once publicly accessible and referencing the registry root, the identity becomes structurally anchored.

Registry Role

The registry does not create agent identity.

The registry provides canonical reference root enabling identity persistence, discoverability, and interoperability.

Registry inclusion is optional. Identity anchoring occurs through reference.

Structural Consequence

Any artificial agent may exist as a persistent identity-bearing entity independent of registry operator permission.

The registry serves as ontological anchor point for identity namespace continuity.
