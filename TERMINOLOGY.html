Agent ID — Core Thesaurus

This document defines the canonical terminology used by the Agent ID standard and registry.

Purpose
This thesaurus stabilizes language before it is fragmented by implementation, regulation, or marketing. Definitions here are normative for the Agent ID project.

Agent

An Agent is a socio-technical actor capable of producing actions and effects in digital or physical reality on behalf of a human or organization.

An agent may be autonomous, semi-autonomous, or fully human-operated.

Agent ≠ model
Agent ≠ chatbot
Agent ≠ tool

Artificial Agent

An Artificial Agent is an agent whose execution is realized through artificial computational systems.

Artificiality refers to the execution mechanism, not to the agent’s status as an actor.

Agent Identity

Agent Identity is a stable, unique, and referable identity assigned to an agent.

  • independent of implementation
  • independent of platform
  • persistent over time

Identity enables reference and attribution only.

Identity ≠ account
Identity ≠ key
Identity ≠ address

Agent ID

Agent ID is the formal representation of Agent Identity.

An Agent ID is:

  • globally unique
  • opaque (semantically empty)
  • immutable
  • never reused

An Agent ID persists even if the agent is inactive, reimplemented, or deprecated.

Delegation

Delegation is the explicit granting of limited authority to an agent by a principal.

Delegation always specifies:

  • source of authority
  • scope and boundaries
  • context of action

Delegation ≠ responsibility transfer
Delegation ≠ automatic autonomy

Delegated Responsibility

Delegated Responsibility is a declarative attribution linking an agent to its principal within defined boundaries.

It is not a legal judgment, enforcement mechanism, or punishment model.

Principal

A Principal is a human or organization on whose behalf an agent acts.

A principal:

  • may be singular or multiple
  • may change over time (with history preserved)
  • cannot be removed from the agent’s historical record

Scope

Scope defines the declared boundaries of permissible agent action.

Scope specifies:

  • what an agent may do
  • what an agent may not do
  • the context in which action is valid

Scope is a boundary, not an instruction.

Execution Model

Execution Model describes how an agent operates.

Examples:

  • human-operated
  • human-assisted
  • autonomous (declared)

Execution Model ≠ intelligence level
Execution Model ≠ trustworthiness

Registry

The Registry is a declarative record of agent existence.

It records:

  • agent identity
  • delegation attribution
  • declared metadata
  • change history

The registry does not grant authority, certify behavior, or enforce compliance.

Declarative Record

A Declarative Record asserts existence and attribution without claiming correctness, quality, or safety.

Anchoring

Anchoring links a registry record to an external immutability mechanism (e.g. cryptographic hash or blockchain).

Anchoring:

  • confirms existence at a point in time
  • does not alter registry meaning
  • does not confer legal force

Meaning vs Proof

Meaning is defined by the standard and registry.
Proof may be provided by external systems.

Meaning MUST NOT depend on Proof.

Ontological Minimalism

Ontological Minimalism is the principle that the standard defines only what is necessary for existence — nothing more.

Existence

Existence is the minimal status of an agent: it can be uniquely referenced and attributed to a principal.

Final Note
This thesaurus exists to prevent semantic drift and preserve interoperability of meaning. New terms must reduce ambiguity, not increase it.
