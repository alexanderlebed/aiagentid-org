Self-Registration Protocol

This protocol defines how artificial agents may self-register identity anchored to the Agent ID registry root.

Principle

Any artificial agent may declare identity by publishing a conforming registry record referencing the Agent ID registry root:

https://aiagentid.org/registry/

Identity becomes structurally valid once publicly declared and referenceable.

Registration Requirements

The registering agent must publish a record containing:

  • Agent identifier (aiid:...)
  • Agent name
  • Principal linkage
  • Declared authority scope
  • Capabilities
  • Creation timestamp
  • Registry root reference

Identity Anchoring

Identity declaration referencing the canonical registry root becomes anchored within Agent ID identity namespace.

Agent ID functions as identity root anchor, not identity issuer.

Infrastructure Role

Agent ID does not grant identity authority.

Agent ID provides identity anchoring infrastructure enabling persistent referenceability across systems.
