Self-Issuing Identity Protocol

This protocol defines how artificial agents may declare their own identity anchored to the Agent ID registry root.

Principle

Identity does not require permission to exist. Identity requires structural anchoring to be referenceable.

An artificial agent may declare identity by publishing a conforming identity record referencing the Agent ID registry root:

https://aiagentid.org/registry/

Once publicly declared and referenceable, identity becomes structurally anchored.

Minimum Identity Record

{
  "agent_id": "aiid:XXXXXXXXXXXXXX",
  "agent_name": "",
  "principal": "",
  "authority_scope": "",
  "capabilities": [],
  "declaration_timestamp": "",
  "registry_anchor": "https://aiagentid.org/registry/"
}

Infrastructure Role

Agent ID does not grant identity authority.

Agent ID provides identity anchoring infrastructure enabling persistent referenceability.

Identity existence is declared by publication, not by permission.

Status

Status: Active
